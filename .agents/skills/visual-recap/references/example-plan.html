    01Overview

    Teams re-apply the same Plans filters every visit. We let them save a filter + sort as a named view and pin it as a tab.

    The Plans list already supports ad-hoc filtering through usePlanFilters(). This work persists a chosen filter set as a saved view: a named, optionally team-shared row in a new saved_views table, surfaced as pinned tabs above the list.

    Goals

    • Save the current filter + sort under a name, private or team-shared.
    • Pin saved views as tabs; deep-link with ?view=<slug>.
    • Reuse the existing filter state — no second source of truth.

    Non-goals

    • Per-column custom layouts or saved sorts outside the list.
    • Cross-workspace sharing (single workspace only for v1).

    02Decisions

    Decision — store the filter state, not a query string

    A saved view stores the structured filter object as jsonb, not a raw URL query. Chosen so the same view survives filter-schema changes and can be validated on read.

    Risk — team-shared views are a new write path

    A shared view is editable by any team member. Guard the update endpoint with the existing canEditWorkspace check and add a test for the cross-user case before shipping.

    Note — filter object already exists

    The client filter shape from usePlanFilters() is reused verbatim as the stored payload, so the model matches what the UI already emits.

    Deferred — reordering pinned views

    Drag-to-reorder tabs is out of scope for v1. Views render in creation order; we revisit ordering once usage data exists.

    03Architecture

    The view resolver reads the saved filter, hydrates the existing filter store, and the list renders unchanged. No new query layer.

    05API

    POST /v1/saved-views Create a saved view

    Request body

    { "name": "My drafts", "shared": true, "filters": { "status": "draft", "owner": "me" } }

    Responses

    201 Created view returned with its slug.
    409 A view with that name already exists.
    GET /v1/saved-views List views visible to the current user

    Responses

    200 Owned views plus team-shared views.
